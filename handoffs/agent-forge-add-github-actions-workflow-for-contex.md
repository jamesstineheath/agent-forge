## Context

Agent Forge needs a GitHub Actions workflow that automatically regenerates and uploads a context snapshot whenever code is merged to `main`. The snapshot is generated by `scripts/generate-context-snapshot.sh` (a pure bash script that already exists or will exist in the repo) and requires `tree` and `jq` system dependencies plus Notion API credentials to upload the result.

This is a simple file-creation task: one new YAML workflow file with no build step, no TypeScript changes, and no dependencies on other files beyond the shell script it invokes.

## Requirements

1. File `.github/workflows/update-context-snapshot.yml` must exist with valid YAML syntax
2. Workflow must trigger on `push` to `main` branch only
3. Workflow must install `tree` and `jq` via `apt-get` before running the script
4. Workflow must pass `NOTION_API_KEY`, `NOTION_CONTEXT_PAGE_ID`, and `GH_TOKEN` as environment variables to the script step
5. Workflow must run `bash scripts/generate-context-snapshot.sh` as its main execution step
6. `GH_TOKEN` must be sourced from `${{ github.token }}` (built-in, no secret needed)
7. `NOTION_API_KEY` and `NOTION_CONTEXT_PAGE_ID` must be sourced from repo secrets

## Execution Steps

### Step 0: Branch setup
```bash
git checkout main && git pull
git checkout -b feat/add-context-snapshot-workflow
```

### Step 1: Create the workflow directory (if it doesn't exist)
```bash
mkdir -p .github/workflows
```

### Step 2: Create the workflow file

Create `.github/workflows/update-context-snapshot.yml` with the following exact content:

```yaml
name: Update Context Snapshot

on:
  push:
    branches: [main]

jobs:
  update-snapshot:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install dependencies
        run: |
          sudo apt-get update -qq
          sudo apt-get install -y -qq tree jq

      - name: Generate and upload context snapshot
        env:
          NOTION_API_KEY: ${{ secrets.NOTION_API_KEY }}
          NOTION_CONTEXT_PAGE_ID: ${{ secrets.NOTION_CONTEXT_PAGE_ID }}
          GH_TOKEN: ${{ github.token }}
        run: bash scripts/generate-context-snapshot.sh
```

### Step 3: Validate YAML syntax
```bash
# If python is available (it will be on most systems)
python3 -c "import yaml; yaml.safe_load(open('.github/workflows/update-context-snapshot.yml'))" && echo "YAML valid"

# Alternative: check with yq if available
# yq '.jobs' .github/workflows/update-context-snapshot.yml
```

### Step 4: Verify file contents look correct
```bash
cat .github/workflows/update-context-snapshot.yml
```

Expected output should show the workflow with:
- `on.push.branches: [main]`
- Three env vars in the final step
- `bash scripts/generate-context-snapshot.sh` as the run command
